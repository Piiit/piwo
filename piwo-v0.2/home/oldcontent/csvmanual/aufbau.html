     5| = Aufbau der CSV-Tabellen =
     6| 
     7| Eine CSV-Datei besteht aus drei Teilen:
     8| * Konfigurationskopf //(Header)//: Die Kopfzeile beinhaltet Kommentare, spezielle [[#Konfigurationen]] und Angaben für Labels von Variablen der [[#Konfigurationskopf (Header)|Datenstruktur]].
     9| * Strukturzeile //(Struct)//: Aufbau der Datenbank (CSV-Datei-Struktur)... Datentypen, Konfigurationen, Spaltennamen, etc.
    10| * Inhalt //(Content)//: Inhalt per CSV (Comma separated Values)
    11|  
    12| Dabei gibt es drei Arten von Zeilentypen:
    13| * Kommentare beginnen mit einem # und können auch am Zeilenende einer Konfigurationszeile stehen (im Contentbereich sind sie nicht erlaubt)
    14| * Konfigurationszeilen beginnen mit !! und haben zwei verschiedene Arten:
    15|  * Form 1: ''!! Schlüssel = Wert'' für einfache Einstellungen oder ...
    16|  * Form 2: ''!! Strukturzeile'' ([[#Strukturzeile (Struct)|siehe Kapitel -> Strukturzeile]])
    17|  
    18| == Konfigurationskopf (Header)==
    19| $$ # HEADER #
    20| $$ # Kommentar: Test für pw_csv                   # Alle Kommentare beginnen mit einem "#"
    21| $$ !! title       = %count Geburtstage            # Konfigurationseinträge beginnen mit "!!" und haben folgende Form "Schlüssel = Wert" (Weitere infos unter pw_config)
    22| $$ !! title_empty = Keine Geburtstage gespeichert # Titel falls keine Einträge in der CSV vorhanden sind
    23| $$ !! title_conf  = Geburtstage merken            # Titel für das Konfigurationsmenü
    24| 
    25| === Konfigurationen ===
    26| | :> ''title'' | Titel der Tabelle, falls Werte vorhanden sind |
    27| | :> ''title_empty'' | Titel der Tabelle, falls der "Content" leer ist |
    28| | :> ''title_conf'' | Titel im Adminbereich der Datenbank |
    29| 
    30| === Labels einzelnen Spalten zuweisen ===
    31| $$ !! datum       = Geburtstag                    # label für die Datenspalte "datum"
    32| 
    33| === Variablen innerhalb der Konfigurationstexte ===
    34| | :> ''%count'' | Anzahl der Datensätze |
    35| | :> ''%'' | ... |
    36| 
    37| == Strukturzeile (Struct)==
    38| 
    39| $$ # STRUCT #
    40| $$ # Nun folgt die Strukturzeile (sie muss in einer Zeile stehen und darf nur einmal vorkommen --> nur die erste Strukturzeile wird geparst, alle anderen ignoriert)
    41| $$ !! &id:int(0,), AUTO_INC|test:int(12,10000)|tage:list(montag, dienstag, mittwoch, donnerstag)|name:csv(test2, name, geschl), NULL |datum:date() | email:text.email(100), NULL
    42| 
    43| 
    44| === Schlüsselwörter ===
    45| | :> ''&''        | bedeutet PRIMARY KEY und ist somit automatisch ''UNIQUE'' |
    46| | :> ''AUTO_INC'' | Werte werden mit jedem neuen Eintrag automatisch inkrementiert |
    47| | :> ''UNIQUE''   | Inhalt darf nur einmal vorkommen (Default = ''NOT UNIQUE'') |
    48| | :> ''NULL''     | bedeutet das Feld darf leer sein (Default = ''NOT NULL'') |
    49| 
    50|  
    51| === Datentypen ===
    52| $$ schlüssel:datentyp(datentypconfig), optionen
    53| oder
    54| $$ schlüssel:datentyp.untertyp(datentypconfig), optionen
    55| 
    56| * Datentypen:
    57|  # :ok: ''text ( länge, default )''
    58|  # :ok: ''date ( format, von, bis, default )'' -> :w:Checks und errorhandling fehlt
    59|  # :ok: ''time ( von, bis, default )'' -> :w:Checks und errorhandling fehlt
    60|  # :ok: ''datetime (von, bis, default )'' -> :w:Checks und errorhandling fehlt
    61|  # :ok: ''int  ( von, bis, default )''
    62|  # :ok: ''float( von, bis, kommazahlen, default )''
    63|  # :ok: ''list ( element1, element2, ... )''
    64|  # ''csv  ( csvname, item1, item2, ... )'' --(es wird immer der "primary key" gespeichert)--
    65|  
    66| * Untertypen: --noch nicht implementiert--
    67|  # ''text.email''
    68|  # ''text.url''
    69|  # ''...''
    70|  
    71| ++Datentyp: DATE++
    72| 
    73| :i: Das Datum innerhalb der CSV hat immer folgende Form: ''JJJJ-MM-TT hh:mm:ss''
    74| $$ date(format, von, bis, default)
    75| Parameterliste:
    76| * **''format''**: Ausgabeformat. --noch nicht implementiert--
    77| * **''von''**, **''bis''**: Zeitraum
    78| * **''default''**: Entweder ein bestimmter Zeitpunkt im CSV-eigenen Datumsformat oder \\ ''NOW'' als Konstante  --(weitere Konstante werden noch implementiert)--
